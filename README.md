## 📖 项目介绍

**PetLink** 是一个以“温馨治愈、情感共鸣”为核心调性的宠物领养与互助平台。  
项目旨在为待领养宠物搭建寻家桥梁，为领养人提供安全可靠的领养渠道，为养宠人打造知识共享与互助交流的社区。

平台聚焦宠物领养全流程服务与养宠互助场景，打造兼具功能性与情感温度的宠物服务平台。

### 🎯 核心定位

- 为救助机构 / 个人救助者提供规范的宠物信息发布渠道
- 为领养人提供安全、透明、可信的领养流程
- 为普通用户提供宠物知识分享与互助交流空间

### 👥 目标用户

| 用户类型 | 描述 |
|---------|------|
| 领养人 | 热爱动物、有养宠意愿，希望通过正规渠道领养宠物的人群 |
| 救助机构 / 个人救助者 | 需要发布待领养宠物信息，寻找可靠领养人的组织或个人 |
| 普通用户 | 关注宠物福利，愿意参与宠物互助、知识交流的爱心人士 |

## ✨ 核心功能模块

### 1️⃣ 用户登录注册模块

- 手机号验证码注册，支持密码登录
- 救助机构 / 个人救助者需完成实名认证（身份证 / 营业执照）
- 账号安全：密码复杂度校验、手机号唯一性校验

### 2️⃣ 领养信息发布管理模块

- 支持发布“待领养宠物”与“寻宠”两类信息
- 内容支持：宠物基础信息、多张照片、短视频
- 差异化字段：健康状况、性格特点、领养要求
- 筛选与排序：按品种、年龄、性别、地区、发布时间、热度等

### 3️⃣ 领养申请流程模块

- 用户可在详情页提交领养申请
- 申请材料：个人信息、居住环境、养宠经验、饲养计划
- 审核机制：救助方审核、平台内置聊天沟通、可选线下家访
- 状态通知：审核通过 / 驳回（附原因）推送通知
- 领养协议：在线签署电子协议，明确双方权利义务

### 4️⃣ 管理员模块

- 独立管理员登录入口 + 二次验证（手机验证码）
- 多级权限管理（超级管理员、内容审核员、用户管理专员）
- 内容审核：信息审核、违规处理、审核规则配置
- 用户管理：查看用户信息、账号封禁、实名复核
- 数据统计：注册用户数、领养信息数、领养成功数、审核通过率等
- 系统配置：审核时效、注册规则、内容规范等

## 📁 数据库设计（核心表结构）

| 表名 | 描述 |
|------|------|
| `user_base` | 用户基础信息（ID、类型、手机号、状态、创建时间） |
| `pet_info` | 宠物信息（品种、类型、描述、状态、媒体、联系方式等） |
| `user_auth` | 用户认证信息（认证类型、媒介、状态） |
| `message_notify` | 消息通知（内容、已读状态） |
| `user_feedback` | 用户反馈（内容、处理状态） |
| `adoption_apply` | 领养申请（关联宠物、用户、申请内容、状态） |
| `adoption_agreement` | 领养协议（关联申请、用户、宠物、协议内容、签署时间） |

> [📎 详细字段说明请参考需求文档中的数据库设计章节。](./宠物领养与互助平台需求文档.docx)

## 🚀 使用方法

本项目为宠物领养与互助平台的**需求文档及系统设计说明**，适用于以下人群：

- 👨‍💻 **开发团队**：参考需求文档进行平台功能开发与数据库设计
- 🎨 **产品经理 / 设计师**：了解平台功能定位与用户流程，进行原型设计
- 📊 **运营人员**：了解平台管理后台功能，制定审核与运营规范

### 📌 使用步骤

1. **阅读需求文档**  
   详细阅读《宠物领养与互助平台需求文档》，理解平台的完整功能需求、用户角色定位及业务流程；参考 ER 图与数据库表结构，完成后端数据模型设计。

2. **分模块开发与测试**  
   按照文档中的功能模块划分（用户端 + 管理员端），依次实现以下核心功能：
   - 用户注册 / 登录 / 实名认证
   - 宠物信息发布与筛选展示
   - 领养申请提交与审核流程
   - 电子协议签署与后续跟踪
   - 管理员后台（内容审核、用户管理、数据统计）
